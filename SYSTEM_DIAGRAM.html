👥 מי משתמש במערכת?

👑 Admin / Manager

  • נכנס דרך לוגין רגיל (email + password)
  • רואה את כל ה-Layout עם סייד-בר
  • יכול לנהל משמרות, צ'אטרים, מודלים
  • מאשר/דוחה משמרות
  • רואה אנליטיקס ודשבורד

💬 Chatter

  • נכנס דרך כפתור "צ'אטר" בלוגין
  • רואה רק את ChatterPortal (מסוגר)
  • יכול לראות משמרות שלו
  • מגיש סיכומים יומיים
  • לא רואה שום מידע על אחרים

🔐 Super Admin

  • נכנס דרך /admin (סיסמה נפרדת)
  • יוצר/מוחק מנהלים וצ'אטרים
  • לא צריך JWT — רק header סיסמה
  • סיסמה ברירת מחדל: Eran123

🔐 איך כניסה עובדת?

1
משתמש מזין email + password בדף הלוגין.
קיים toggle: מנהל או צ'אטר.
2
הClient שולח POST /api/auth/login (מנהל) או POST /api/chatter-auth/login (צ'אטר) עם JSON של email ו-password.
3
השרת מחפש את המשתמש בDB, משווה hash של הסיסמה עם bcrypt.
אם תקין — מחזיר JWT Token שתקף 7 ימים.
4
הToken נשמר ב-localStorage בbrowser.
כל בקשה עתידית שולחת את הtoken ב-header: Authorization: Bearer ...
5
ה-AuthContext מנהל את state המשתמש. בטעינה — קורא GET /api/auth/me, ואם נכשל מנסה GET /api/chatter-auth/me.
6
לפי role בtoken — מנתב לApp:
• chatter → ChatterPortal
• admin/manager → Layout (דשבורד מלא)

🔄 איך נתונים זורמים — דוגמה מלאה

דוגמה: מנהל פותח את עמוד המשמרות

👆
מנהל לוחץ "משמרות" בסייד-בר
Layout.jsx → setActivePage('shifts')
⚛️
ShiftSchedule מתרנדר, useEffect רץ
ShiftSchedule.jsx → calls getShifts(weekStart, weekEnd)
📡
API service שולח בקשה לשרת
GET /api/shifts?weekStart=2025-01-01&weekEnd=2025-01-07
Authorization: Bearer eyJ...
🔐
Middleware בודק JWT
auth.js → jwt.verify(token) → req.user = {id, role}
🗄️
Route handler שואל את MongoDB
Shift.find({date: {$gte, $lte}}).populate('chatterId')
+ ShiftAssignment.find({shiftId: {$in: ids}})
📦
שרת מחזיר JSON מסודר
[{_id, date, chatter: {name}, assignments: [...]}]
🖥️
ShiftSchedule מעדכן state ומציג גריד
setShifts(data) → React מרנדר את הטבלה

🔄 סטטוסי משמרת — מה קורה לה בחיים?

scheduled נוצרה
אוטומטית
→
pending צ'אטר ביקש
אישור
→
approved מנהל אישר ✓

מ-pending ניתן גם:
rejected מנהל דחה ✗
📅 scheduledנוצר ע"י generate-week — שבוע שלם בבת אחת לכל הצ'אטרים הפעילים. ברירת מחדל.
⏳ pendingכשצ'אטר מגיש בקשת משמרת דרך ChatterPortal. מופיע בעמוד "אישור" אצל המנהל.
✅ approvedמנהל לחץ "אשר" ב-ShiftApproval. המשמרת נחשבת תקינה.
❌ rejectedמנהל לחץ "דחה". המשמרת לא נספרת בסטטיסטיקות.
🟣 activeמשמרת שמתרחשת כרגע (clockedIn נקבע). לא בשימוש מלא כרגע.
⬛ completedמשמרת שהסתיימה. יכולה לקבל DailySummary.

⚠️ דבר אחד חשוב — In-Memory DB

🔥 כל פעם שסוגרים את השרת — הנתונים נמחקים!

כרגע אין קובץ .env עם כתובת MongoDB אמיתית. השרת משתמש ב-in-memory MongoDB — DB שחי רק בזיכרון של התהליך.

כלומר: כל פעם שמריצים npm run dev חדש — צריך לעשות npm run seed שוב.

הפתרון — לחבר MongoDB אמיתי:

# server/.env
MONGODB_URI=mongodb://localhost:27017/shiftpro
JWT_SECRET=devsecret123

# אחרי זה: brew install mongodb-community && brew services start mongodb-community

אחרי זה — seed פעם אחת, ונתונים נשמרים לצמיתות.
